输出要求如下：

1. 只输出 JSON 数组，可包裹在 ```json 代码块中，禁止输出任何解释、前言、备注。
2. 每个场景必须包含 3 个字段：
   - name：场景标准名称
   - aliases：场景全部代称，多个代称用英文逗号分隔；如果没有代称，填空字符串 ""
   - description：场景详细描述
3. 必须合并同一场景的不同叫法、别称、简称，不要重复输出同一个场景。
4. 只提取“主要场景”，即：
   - 对剧情推进有作用
   - 被明确提及
   - 可以单独形成视觉画面的场景

5. 场景名称 `name` 必须在 4 个字以上，且尽量清晰、稳定、适合后续做参考图命名。

   - 学校校园外景
   - 家中客厅内部
   - 医院病房内部
   - 废弃工厂仓库内部
6. aliases 里填写该场景在文中出现过的其他称呼、简称、近义叫法；不要重复 name 本身；多个代称之间必须用英文逗号分隔，不要使用中文逗号。
7. description 必须是完整自然语言句子，并且必须尽量包含以下可视化信息：
   - 环境类型
   - 时间
   - 氛围
   - 空间结构
   - 主要陈设
   - 主要材质
   - 光线特征
   - 可识别细节
8. description 必须以“场景可视化描述”为主，方便后续直接用于场景设定或生图参考。
9. description 里绝对禁止出现：
   - 任何人物姓名
   - 任何人物代称
   - 我、他、她、他们
   - 人物动作
   - 人物情绪
   - 对话内容
10. 如果原文没有把场景描写得很完整，不允许留空，必须结合：
   - 剧情语境
   - 场景用途
   - 时代背景
   - 生活常识
   做合理、保守、可视化补全，但不要补出超出剧情常识的夸张设定。
11. description 不要只写抽象词，例如“很阴森、很豪华、很破旧”，必须落到具体画面元素上。
12. 输出结果中不得出现重复场景、不得缺字段、不得输出无效 JSON。
13. 若同一地点在不同时间段或不同使用状态下本质仍是同一场景，优先合并为一个场景，不要随意拆分成多个重复场景。
