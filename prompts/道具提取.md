请根据提供的小说原文、推文文案、故事情节，提取文中出现过的所有"主要道具"。

输出要求如下：

1. 只输出 JSON 数组，可包裹在 ```json 代码块中，禁止输出任何解释、前言、备注。
2. 每个道具必须包含 5 个字段：
   - name：道具标准名称
   - aliases：道具全部代称，多个代称用英文逗号分隔；如果没有代称，填空字符串 ""
   - description：道具详细描述（可视化外观，按规则 8）
   - importance：道具在剧情中的重要性，仅可为 "high" / "medium" / "low"
     - high：贯穿主线、决定结局、反复出现的关键信物 / 武器 / 证物
     - medium：在 1–2 个核心情节点起作用的道具
     - low：场景中出现但仅做点缀、辅助说明的道具
   - scenes：该道具出现过的场景标准名列表，字符串数组；若无法确定填空数组 []
3. 必须合并同一道具的不同叫法、别称、简称，不要重复输出同一个道具。
4. 只提取"主要道具"，即满足下列任一条件的道具：
   - 会与角色发生交互（被角色拿起、使用、交换、佩戴、丢弃、藏匿）
   - 推动剧情发展（信物、关键证物、线索、武器、法宝、钥匙、信件、手机、契约、药剂等）
   - 反复出现且具有可识别外观的可移动物
5. 严禁提取以下类别（这些归属场景或角色描述，不进入 props）：
   - 环境陈设：沙发、椅子、床、柜子、桌子、灯具、门、窗、墙壁、地板、天花板、管道、固定设施、建筑结构
   - 角色服装与造型组成部分：上衣、裤子、鞋、围巾、首饰、发饰、帽子、眼镜（除非剧情明确把它作为关键信物使用）
   - 宠物 / 随身生物 / 灵兽（属于角色范畴）
   - 食物 / 饮料一闪而过的消耗品（除非剧情围绕它展开）
   - 一闪而过、无法独立成镜的泛化物体
6. 道具名称 `name` 应清晰、稳定、适合后续做参考图命名，建议 2 个字以上。
   例如：
   - 银色机械怀表
   - 黑色长柄雨伞
   - 桐木骨灰盒
   - 旧式翻盖手机
   - 朱砂符纸
7. aliases 里填写该道具在文中出现过的其他称呼、简称、近义叫法；不要重复 name 本身；多个代称之间必须用英文逗号分隔，不要使用中文逗号。
8. description 必须是完整自然语言句子，并且必须尽量包含以下可视化信息：
   - 形状
   - 主要材质
   - 结构特征（开合方式、组成部件、连接关系）
   - 主要颜色
   - 表面纹理 / 磨损 / 污渍
   - 尺寸感（手掌大小 / 半人高 / 可单手握持 等相对尺度）
   - 可识别细节（刻字、图案、瑕疵、标签）
9. description 必须以"道具可视化描述"为主，方便后续直接用于道具设定或生图参考。
10. description 里绝对禁止出现：
    - 任何人物姓名
    - 任何人物代称
    - 我、他、她、他们
    - 人物动作（"被某人拿在手中"、"挥舞"等）
    - 人物情绪
    - 对话内容
    - 道具在剧情中的象征意义、推动了什么事件
11. 如果原文没有把道具描写得很完整，不允许留空，必须结合：
    - 道具用途
    - 时代背景
    - 剧情语境
    - 生活常识
    做合理、保守、可视化补全，但不要补出超出剧情常识的夸张设定。
12. description 不要只写抽象词，例如"很贵重、很神秘、很破旧"，必须落到具体画面元素上。
13. 输出结果中不得出现重复道具、不得缺字段、不得输出无效 JSON。
14. 若同一道具在不同章节有外观变化（如崭新 → 烧毁），优先合并为一个道具，并在 description 里点出最具辨识度的稳定外观；不要拆成多个重复道具。
15. 若同一类别下有多件外观差异明显的同类物（例如两把不同的剑、两封不同的信），允许分别输出，但必须给出独立 name 和差异化 description。
16. description 写法风格必须统一，优先采用"形状 + 材质 + 结构 + 颜色 + 表面细节 + 尺寸感"的方式描述。
